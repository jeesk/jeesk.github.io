REPETABLE READ

案例3：

B 用户 开启一个事务后，一直读取 ID = 1 的数据。

A 用户 修改 ID = 1 的这一条数据

B 用户继续查询 ID = 1 的这一条数据,但是不提交数据。

你会发现， 在当前事务, 及时另外一个事务修改了数据已经提交事务，但是读到的数据产生了幻觉。 这就叫可以重复读。也叫幻读。

SERIALIZABLE

串行化， 以上3种隔离级别都允许对同一条记录进行读-读、读-写、写-读的并发操作，如果我们不允许读-写、写-读的并发操作，可以使用SERIALIZABLE隔离级别，示意图如下：

当用户 A中的事务更新了id为1的记录后，之后用户 B中的事务再去访问这条记录时就被卡住了，直到用户 A中的事务提交之后，用户 B中的事务才可以获取到查询结果。 发现，这 SERIALIZABLE 这种事务模式下面的效率非常低， 因为当某条数据被事务占用的时候， 另外的连接无法修改，并且连查询也不能。
